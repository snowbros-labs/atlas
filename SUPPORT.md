# Support

Thanks for using Snowbros Atlas. Here is the fastest path to help.

## I have a question or want to discuss an idea

Open a [Discussion](https://github.com/snowbros-labs/atlas/discussions). Use it
for usage questions, ideas, and showing off what Atlas found in your repo.

## I found a bug or a false positive

Open a [bug report](https://github.com/snowbros-labs/atlas/issues/new?template=bug_report.yml).
A minimal reproduction (a few files, the command you ran, the output you got and
expected) is the single most useful thing you can include.

False positives matter to us specifically: Atlas only ships findings it can
prove, so a false positive is a real defect. Please include the rule id and the
code that triggered it.

## I want a new rule or feature

Open a [feature request](https://github.com/snowbros-labs/atlas/issues/new?template=feature_request.yml).
For rules, read [docs/adding-a-rule.md](docs/adding-a-rule.md) first — and
consider contributing it.

## I found a security issue

Do **not** open a public issue. Follow [SECURITY.md](SECURITY.md) — report
privately via GitHub Security Advisories.

## Response expectations

Snowbros Atlas is an open-source project maintained on a best-effort basis.
There is no paid support tier and no SLA. Security reports follow the timelines
in [SECURITY.md](SECURITY.md); everything else is triaged as time allows, with
help from the community.
