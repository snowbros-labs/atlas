Features

Whole-project semantic graph

SNOWBROS scans every file, parses JS/TS/JSX/TSX with Tree-sitter, and resolves every import — relative paths, tsconfig.json path aliases (including extends chains), TypeScript-ESM ./file.js → file.ts substitution, and re-export chains. The result is a graph it can query for cycles (Tarjan SCC), dead files (reverse reachability from entry points), and boundary violations.

Next.js server/client boundary tracking

The flagship rule, next/server-only-in-client, walks the graph from every client component and reports server-only code it can reach — through aliases and re-exports — printing the exact import chain as evidence. Companion rule next/private-env-in-client catches non-NEXT_PUBLIC env vars read in client code.

Deterministic auto-fix

sb fix plans byte-span edits from findings, then applies them only if the file still matches what analysis saw. Unused dependencies are removed with format-preserving package.json surgery; unused env vars with guarded line deletion. Idempotent, proven by end-to-end tests. --dry-run previews everything.

Editor and CI integration

  • LSP: sb lsp serves diagnostics over stdio to any LSP-capable editor; re-analyzes on open/save in tens of milliseconds warm.
  • SARIF 2.1.0: upload to GitHub code scanning; findings appear in the Security tab.
  • CI gate: sb analyze --ci exits 2 on High+ findings.
  • Watch mode: sb watch prints only new and resolved findings as you edit.

Explainable health score

Findings roll up into a 0–100 scorecard across six categories with documented weights (severity 25/15/8/3/1 × confidence 1.0/0.7/0.4/0.2) — every point of deduction traceable to a finding.

Config that means what it says

[analysis]
min_severity = "low"
min_confidence = "possible"

[rules]
disable = ["exports/*"]      # category globs
enable  = ["exports/unused-export"]  # enable wins

An invalid config is a hard error, never silently ignored.
