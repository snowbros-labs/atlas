Contributing

Full guide: CONTRIBUTING.md. The short version:

The ground rules

  1. Determinism is non-negotiable. Findings are a pure function of code + config. No timestamps, no unstable iteration order, no network during analysis.
  2. Accuracy over quantity. Unprovable → unresolved, never guessed. False-positive guards documented per rule.
  3. Evidence is mandatory. Every diagnostic carries its chain.
  4. The cache may never change results. Warm == cold, byte-identical, enforced by tests.
  5. Secrets are always redacted.

Getting started

git clone https://github.com/snowbros/snowbros-inspector
cd snowbros-inspector
cargo test --workspace
cargo clippy --workspace --all-targets -- -D warnings

Best first contributions

  • False-positive reports with a minimal repro — the most valuable input at this stage.
  • A new rule with metadata + FP-guard tests (see the rule-authoring checklist in CONTRIBUTING.md).
  • Resolver edge cases from real projects (monorepos, package.json#exports).

Commits follow Conventional Commits; the changelog is generated from them. Contributions are dual-licensed MIT OR Apache-2.0.
