Roadmap

Direction, not promises. Ordering reflects current priorities; 0.x minor versions may include breaking CLI/output changes per SemVer.

Near term

  • Pattern rule engine — Semgrep-style $VAR patterns in YAML, so new rules don't require Rust.
  • More rules toward 20–50: N+1 query detection, oversized components, prop drilling, hydration hazards, unprotected routes, SEO/a11y structure.
  • Resolver: package.json#main/exports for directory and package-root imports (removes the fastify-style unresolved findings).
  • Monorepo awareness — multiple package.json workspaces as first-class projects.
  • sb doctor and rule maturity gating (nursery rules off by default).

Mid term

  • Symbol-level call graph — resolution below file granularity.
  • Plugins — WASM (Extism) rule plugins.
  • More languages — Python and Go grammars.
  • Vulnerability DB — OSV integration for dependency advisories.

Long term

  • Trend dashboard, multi-repo views.
  • Optional AI explanation layer — AI may explain findings, never decide them. Determinism stays non-negotiable.
